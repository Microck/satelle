# Specification Gaps

This ledger records behavior that existing facts do not define and that blocks implementation or verification. It must not add product scope or substitute for `.facts`.

## Open Gaps

### GAP-004 - Release attestation trust policy

Description: Release facts require attestations and verification but do not define the complete trust policy.
Component: Release attestation and verification
Exposing fact or work unit: Attestation generation, archive verification, TUF trust roots, and signed-tag verification
Undefined behavior: Predicate type, archive versus raw-binary subject, verifier mechanism, TUF bootstrap and root-update policy, and the signed-tag trust source.
Blocking: yes

### GAP-005 - Installer and package-manager lifecycle contract

Description: Distribution facts do not close the native installer and package-manager lifecycle across supported platforms.
Component: Native installers, Homebrew, Scoop, npm, pnpm, and Bun distribution
Exposing fact or work unit: Installer validation and release publication gates
Undefined behavior: Canonical URLs, arguments, destinations, receipt schema and lifecycle, upgrade and uninstall semantics, failure atomicity, Linux and Windows 11 architecture matrix, tap and bucket identities, ownership detection, and release sequence.
Blocking: yes

### GAP-006 - Embedded skill bundle CLI contract

Description: Skill-bundle facts do not define the observable CLI and materialization contract.
Component: Embedded agent skill distribution
Exposing fact or work unit: `skills get`, `skills path`, version output, and public command examples
Undefined behavior: Output formats, path materialization semantics, bundle membership including `SKILL.md` and `agents/openai.yaml`, version output, and command-example scope.
Blocking: yes

### GAP-007 - Recovery suggestion command contract

Description: Facts do not establish that every emitted recovery suggestion is a complete executable command.
Component: CLI, MCP, setup, doctor, and completion recovery output
Exposing fact or work unit: Canonical `suggested_commands` verification
Undefined behavior: Whether prose is permitted, the canonical replacement for invalid `host trust`, supported doctor storage scope, and whether completion placeholders may be emitted as commands.
Blocking: yes

### GAP-008 - Public documentation and support boundary

Description: Public documentation facts do not close the version, deployment, support, and vulnerability-reporting contract.
Component: README, Fumadocs, SECURITY.md, and release documentation
Exposing fact or work unit: Public documentation and release-readiness gates
Undefined behavior: Authoritative release version, versioned routes and deployment target, Node.js floor, public source roots, private vulnerability-reporting channel, and Windows 11 support boundary.
Blocking: yes
