   - Lane A: Host, Bridge, Sessions, Storage, then Computer Use.
   - Lane B: CLI, Errors, then Config.
   - Lane C: Release, Docs, npm/install, then attestation.
5. Phase 4: run real-environment verification and release gates, then audit the definition of done.

PRs may develop in parallel, but reviews, merges, and main synchronization are serialized. Each major work unit uses a fresh semantic branch, relevant tests, a push with Residue hooks intact, a PR, Greptile and Codex review when available, fixes for valid findings, merge, and a main sync before the next merge.
