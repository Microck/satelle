Global options include `--no-color`, `--profile <name>`, and
`--error-format human|json`. Most state-reading commands also accept `--json`.

| `satelle repair` | Repair a Host | Not implemented |
| `satelle doctor` | Run diagnostic probes | Implemented |
| `satelle config check` | Validate one or all local contexts | Implemented |
| `satelle config explain` | Explain effective values and sources | Implemented |
| `satelle paths` | Print resolved config, state, cache, and log paths | Implemented |
| `satelle host start` | Run Host Daemon mode | Implemented |
| `satelle host trust --host <alias>` | Authenticate a direct Host and pin its stable identity | Implemented |
| `satelle host status` | Read Host status | Implemented |
| `satelle host sessions` | List eligible native desktop sessions | Implemented |
| `satelle host stop` / `restart` | Control Host service lifecycle | Not implemented |
| `satelle host update` | Update remote components | Not implemented |
| `satelle host storage migrate` | Run an explicit storage migration | Not implemented |
| `satelle run` | Create a Session and first Turn | Implemented |
| `satelle steer` | Add a follow-up Turn to a Session | Implemented |

| `satelle stop` | Stop the active Turn | Implemented |

`setup` application is transport-specific. Local setup is dry-run planning
only. SSH setup can apply the on-demand transport token handoff only. Direct
setup, persistent Host service installation, and SSH setup for broader
components are unavailable.

Direct and SSH-tunneled transports support ordinary Session operations. Live
`doctor` probes are currently local-only; unsupported transport/scope
combinations fail with a typed error.

Use `satelle <command> --help` as the exact option reference for the binary you
are running. Pre-release commands may change before the first stable release.
The [generated CLI and schema reference](/docs/reference/generated-cli) records
the complete help tree and public schema identifiers for Satelle 0.1.0.

## Session commands

`run` and `steer` accept `--host`, `--detach`, `--quiet`, `--verbose`,
`--events auto|human|json|none`, `--prompt-file`, `--yolo`, and `--no-yolo`.
The prompt can be an argument or `-` for stdin. `steer`, `status`, and `stop`
take a public `session_id` positional argument.

When experimental provider Computer Use is enabled, attached `run` and `steer`
JSON results include `provider_smoke` with `status`, `source`, `observed_at`,
`expires_at`, and `age_ms`. The source is `live`, `cache`, or `refresh`.
`--refresh-provider-smoke-test` forces `refresh` before prompt execution.
Provider fingerprints, prompts, transcripts, and desktop content are excluded.

## Doctor options

`doctor` accepts `--host`, `--scope`, `--refresh`, `--timeout`,
`--serial-probes`, `--events`, and `--no-input`. `--timeout` is valid for an
eligible refresh scope and requires explicit units.

`doctor --scope provider --refresh` runs the provider smoke probe as a
diagnostic, updates the normal Host readiness cache, and reports whether the
result was refreshed. It does not create a Session or admit a prompt Turn.

## Unavailable command families

Commands can appear in `--help` before their implementation is complete.
`repair`, Host stop/restart/update, storage migration, self-update, and support
bundle export currently return typed not-implemented errors. There is no
`satelle skills` command or embedded Agent Skill Bundle in this source tree.
