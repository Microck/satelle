---
title: Run your first session
description: Build Satelle, inspect setup, prove native readiness, and start durable work.
---

This tutorial uses `local-demo`, the built-in local Host binding. Run it on the
macOS or Windows machine whose visible desktop Satelle will control.

## 1. Build the executable

Install Rust 1.97.0 and Codex 0.144.0, then build from the repository root:

```sh
cargo build --release --locked -p satelle-cli
```

Use `target/release/satelle` below, or put that executable on your `PATH`.
Public npm packages are not available yet.

## 2. Inspect setup before changing anything

```sh
satelle paths
satelle config check
satelle setup --host local-demo --dry-run
```

The dry run reports planned actions, required human input, readiness, and
recovery commands. Local setup mutation is not implemented in this release, so
do not rerun this local command with `--yes`. Complete any reported Host,
runtime, desktop, or provider prerequisites through operator-controlled tools.
Satelle does not silently answer operating-system permission prompts.

## 3. Prove native readiness

Keep the target desktop unlocked and visible, then run:

```sh
satelle doctor --host local-demo --scope computer-use --refresh
```

On macOS, grant Screen Recording and Accessibility when the operating system
asks. On Windows, keep the target app on the active desktop. Satelle does not
click through these prompts for you.

## 4. Start durable work

```sh
satelle run --host local-demo "Open the browser"
```

Capture the returned `session_id`. It is the public handle for follow-up work,
status, logs, and stop requests.

```sh
satelle status <session_id>
satelle logs --session <session_id> --tail 50
```

## 5. Reconnect and control the Session

Close the original shell or start a fresh Controller process. Confirm the
Session still exists, start a detached follow-up Turn, then stop it:

```sh
satelle status <session_id> --host local-demo
satelle steer <session_id> --host local-demo --detach "Open settings"
satelle stop <session_id> --host local-demo
satelle status <session_id> --host local-demo
```

You have completed the tutorial when the first Turn reached a terminal state,
the fresh Controller read the same Session, and the final status confirms that
the detached follow-up Turn stopped.

## Current limitations

- Native Host readiness is supported only on candidate macOS and Windows
  Hosts that pass the live probe. Linux native Computer Use is unavailable.
- The release has not been published to npm.
- SSH on-demand transport setup and tunneled operation are implemented, but
  persistent remote service installation is not.
- Local setup mutation, direct setup, automatic repair, and support bundle
  export are not implemented.
